[FIXED] Contacts not uploading / job stuck in "pending" forever
  PREVIOUS SYMPTOM:
    - Import returned 200 OK but contacts never appeared.
    - Frontend polled GET /contacts/jobs/{job_id} every 2s indefinitely.
    - RabbitMQ published the task but nothing consumed it.

  ROOT CAUSE:
    The csv_import task was published to a RabbitMQ queue ("background_tasks")
    that required a separate background_worker.py process to be running.
    If that process was not started, the job stayed in status="pending" forever.

  ARCHITECTURAL FIX (applied 2026-04-04):
    Removed the RabbitMQ dependency for CSV imports entirely.
    Contact imports now run via FastAPI BackgroundTasks inside the API process.
    As long as the API server (Terminal 1) is running, imports will always work.

    Changed files:
      platform/api/services/import_service.py   [NEW] — import processing logic
      platform/api/routes/contacts.py            [MODIFIED] — uses BackgroundTasks
        instead of mq_client.publish_background_task()

    background_worker.py is now OPTIONAL (kept for future RabbitMQ tasks).
